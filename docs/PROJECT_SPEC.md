# Project Specification

이 문서는 `english_memory_note_maker`의 현재 사양 명세 초안입니다.

## 1. 프로젝트 개요

- 프로젝트 목적:
  웹페이지에서 여러 장의 이미지 파일을 업로드받아, 그 안의 영어 단어와 뜻을 추출하고 학습용 암기 노트를 자동으로 생성해 `PDF`와 `XLS`로 내려받게 한다.
- 해결하려는 문제:
  이미지 형태로 흩어져 있는 단어 학습 자료를 사람이 직접 다시 정리하지 않아도 웹에서 업로드만으로 단어장 형태로 재구성할 수 있게 한다.
- 기대 효과:
  단어 학습 자료 정리 시간을 줄이고, 웹에서 바로 확인하고 `A4` 기준으로 활용 가능한 암기 노트 파일을 만든다.

## 2. 대상 사용자

- 주요 사용자:
  영어 단어를 암기하려는 학습자, 직접 단어장을 만드는 학생 또는 교사
- 사용 환경:
  웹브라우저에서 이미지 파일을 업로드하고 결과물을 확인한 뒤 `PDF` 또는 `XLS`로 내려받아 인쇄하거나 검토하는 환경
- 사용자 수준:
  복잡한 편집 없이 파일을 넣고 결과를 받길 원하는 일반 사용자

## 3. 핵심 사용자 시나리오

1. 사용자는 웹페이지에서 영어 단어와 뜻이 적힌 `jpg` 또는 `png` 이미지 파일 여러 개를 업로드한다.
2. 시스템은 업로드된 이미지에서 영어 단어와 뜻을 읽어내고 이를 단어-뜻 목록으로 정리한다.
3. 시스템은 정리된 목록을 바탕으로 암기 노트를 생성한다.
4. 사용자는 웹페이지에서 생성된 단어장을 확인한다.
5. 사용자는 `뜻 쓰기`와 `단어 쓰기`를 각각 몇 차까지 반복할지 선택한다.
6. 사용자는 암기 노트를 `PDF` 또는 `XLS`로 다운로드한다.
7. 사용자는 출력된 암기 노트에 직접 뜻 또는 단어를 작성하며 학습한다.
8. 사용자는 문서 마지막의 정답지로 원본 순서 기준 정답을 확인한다.

## 4. 기능 요구사항

### 필수 기능

- [x] 웹페이지에서 여러 개의 `jpg` 또는 `png` 이미지 파일을 업로드할 수 있어야 한다.
- [x] 업로드된 이미지 안의 영어 단어와 뜻을 추출할 수 있어야 한다.
- [x] 추출한 내용을 단어-뜻 쌍으로 구조화할 수 있어야 한다.
- [x] 웹페이지에서 생성된 단어장을 확인할 수 있어야 한다.
- [x] 단어만 제시하고 뜻을 쓰게 하는 암기 노트를 생성할 수 있어야 한다.
- [x] 뜻만 제시하고 단어를 쓰게 하는 암기 노트를 생성할 수 있어야 한다.
- [x] 두 가지 형식의 결과를 하나의 학습용 결과물로 제공할 수 있어야 한다.
- [x] 사용자가 결과를 `PDF` 또는 `XLS`로 다운로드할 수 있어야 한다.
- [x] 출력 결과는 사용자가 손으로 답을 적을 수 있도록 충분한 작성 공간을 제공해야 한다.
- [x] 문제 번호는 원본 추출 순서를 유지해야 한다.
- [x] 문서 마지막에 정답지를 포함해야 한다.
- [x] `뜻 쓰기`와 `단어 쓰기`를 각각 반복 회차 기준으로 여러 차시 생성할 수 있어야 한다.
- [x] 반복 회차를 `0`으로 두면 해당 유형을 제외할 수 있어야 한다.

### 선택 기능

- [ ] OCR 오인식 내용을 사용자가 수정할 수 있는 기능
- [ ] 중복 단어 제거 또는 병합 기능
- [ ] 난이도 또는 알파벳 순 정렬 기능
- [ ] 출력 레이아웃 세부 프리셋 선택 기능

## 5. 입출력 정의

- 입력 데이터:
  웹페이지를 통해 업로드된 영어 단어와 뜻이 포함된 `jpg`, `png` 형식의 이미지 파일 여러 개
- 출력 결과물:
  웹페이지에서 확인 가능하고 `PDF` 또는 `XLS`로 다운로드 가능한, 단어 쓰기용 섹션과 뜻 쓰기용 섹션 및 정답지가 포함된 암기 노트
- 저장 형식:
  현재는 `PDF`와 `XLS`

## 6. 비기능 요구사항

- 성능:
  여러 장의 업로드 이미지를 안정적으로 처리할 수 있어야 한다.
- 사용성:
  복잡한 설정 없이 웹페이지에서 파일 업로드 후 결과를 생성할 수 있어야 한다.
- 유지보수성:
  업로드 처리, OCR, 데이터 정리, 출력 생성 로직이 분리된 구조여야 한다.
- 호환성:
  일반적인 데스크톱 및 모바일 브라우저에서 사용 가능해야 하며, `jpg`, `png` 이미지 파일을 처리할 수 있어야 한다.
- 인쇄 적합성:
  결과물은 `A4` 인쇄를 기준으로 표 레이아웃과 헤더 반복이 유지되어야 하며, 학습용 작성 공간이 유지되어야 한다.

## 7. 제약사항

- 기술 제약:
  이미지 안의 텍스트 품질과 배치 형태에 따라 인식 정확도가 달라질 수 있으며, 웹 업로드 환경에 맞는 파일 크기 제약이 필요할 수 있다.
- 일정 제약:
  현재는 초기 기능 구현과 문서 정합화 단계다.
- 운영 제약:
  입력 이미지의 해상도와 글자 선명도에 결과 품질이 크게 영향을 받는다.

## 8. 미정 항목

- [ ] 단어와 뜻이 이미지 안에서 어떤 배치 규칙을 따르는지
- [ ] OCR 결과를 사용자가 직접 수정할 수 있게 할지
- [ ] 회차별 셔플 규칙을 사용자 옵션으로 노출할지
- [ ] PDF와 XLS의 정답지 포함 여부를 별도 선택하게 할지
- [ ] OCR 결과를 사용자가 검수하는 단계가 필요한지
- [ ] 여러 이미지에서 추출한 항목의 중복 처리 기준
