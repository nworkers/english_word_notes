## 현재 동작 방식

- 사용자가 웹페이지에서 이미지 파일 여러 개를 선택한다.
- 클라이언트가 `FormData`로 `/api/extract`에 업로드 요청을 보낸다.
- 서버 API는 업로드된 파일 목록을 받아 목 추출 결과를 만든다.
- 결과 화면은 단어를 보고 뜻을 쓰는 섹션과 뜻을 보고 단어를 쓰는 섹션을 함께 렌더링한다.
- 브라우저의 인쇄 기능으로 결과 화면을 출력할 수 있다.

## 아직 구현되지 않은 부분

- 실제 OCR 엔진 연동
- OCR 결과 보정 및 검수 기능
- 업로드 파일 저장 정책
- 결과 영구 저장
- 인쇄 레이아웃의 페이지 단위 최적화

## 결정 사항

- 프레임워크 초안은 `Next.js + TypeScript`로 시작
- OCR은 이후 교체 가능한 별도 로직으로 분리
- MVP 단계에서는 목 데이터로 전체 흐름을 먼저 연결
