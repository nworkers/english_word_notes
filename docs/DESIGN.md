# Design Notes

이 문서는 이미지 기반 영어 단어 자료를 웹에서 업로드받아 암기 노트로 변환하고 `PDF` / `XLS`로 내보내기 위한 설계 문서입니다.

## 1. 목표 구조

- 애플리케이션 형태:
  웹페이지에서 여러 이미지 파일을 업로드받고, 서버 또는 백엔드 처리 후 결과를 화면에 보여주는 웹 서비스
- 주요 실행 흐름:
  이미지 업로드 -> OCR 추출 -> 단어/뜻 구조화 -> 암기 노트 생성 -> 웹 화면 미리보기 -> `PDF` / `XLS` 내보내기
- 결과물 생성 방식:
  단어 쓰기용 섹션과 뜻 쓰기용 섹션을 포함한 학습용 결과를 웹페이지에서 보여주고, OCR 결과 기반 제목이 포함된 `A4` 기준 표 레이아웃의 `PDF` / `XLS` 파일로 생성

## 2. 구성 요소

- UI 또는 입력 인터페이스:
  여러 이미지 파일을 업로드할 수 있는 웹 업로드 화면, 처리 상태 로그/진행률 표시 영역, 팝업 기반 `Ollama` / `Gemini` / `OpenAI` 설정 저장 영역
- 핵심 로직:
  OCR 결과에서 영어 단어와 뜻을 구분하고 쌍으로 매핑하는 로직
- 데이터 저장소:
  업로드된 파일 정보와 추출된 단어 목록을 메모리 또는 중간 구조 데이터로 관리
- 출력 생성기:
  학습용 노트 형식으로 재구성해 웹 미리보기와 `PDF` / `XLS` 출력용 레이아웃을 만드는 렌더링 모듈

## 3. 주요 화면

- 업로드 화면:
  사용자가 여러 이미지 파일을 선택하고 업로드를 시작하는 화면
- 결과 화면:
  추출된 단어장과 암기 노트를 확인하는 화면
- 출력 화면:
  별도 인쇄 페이지 없이, 홈 화면에서 반복 회차를 설정하고 `PDF` / `XLS`를 바로 다운로드하는 흐름

## 4. 데이터 흐름

1. 사용자가 웹페이지에서 `jpg` 또는 `png` 이미지 파일 여러 개를 업로드한다.
2. 업로드 처리 모듈이 파일을 수신하고 OCR 처리 대상으로 전달한다.
3. OCR 모듈이 각 이미지에서 텍스트를 읽어낸다.
4. 파서가 텍스트에서 영어 단어와 뜻의 관계를 분석한다.
5. 정규화 단계에서 공백, 중복, OCR 오인식 가능 항목을 정리한다.
6. 원본 이미지에 번호가 보이면 이를 우선 보존하고, 챕터별 번호 재시작은 순차 번호로 보정한다.
7. 출력 모듈이 다음 두 섹션을 포함한 암기 노트를 생성한다.
   - 영어 단어 제시 후 뜻을 쓰는 섹션
   - 뜻 제시 후 영어 단어를 쓰는 섹션
8. 사용자는 각 섹션의 반복 회차를 따로 지정할 수 있다.
9. 결과 화면은 구조화된 추출 결과를 번호순 JSON으로 보여주고, 정답지는 별도 번호표 섹션으로 분리해 보여준다.
10. 사용자는 설정 팝업을 통해 브라우저 내부에 `Ollama` / `Gemini` / `OpenAI` 설정을 저장할 수 있다.
11. `Ollama` 설정은 로컬 서버와 `Ollama Cloud`를 모두 수용할 수 있어야 한다.
12. 처리 중에는 단계 로그, 진행률 바, 파일 처리 개수, 남은 단계가 갱신된다.
13. 사용자는 결과 화면에서 자동 생성된 내보내기 제목을 수정할 수 있다.
14. 내보내기 모듈이 원본 순서 번호와 정답지를 포함한 `PDF` / `XLS`를 생성한다.
15. 사용자는 웹에서 결과를 확인하고 파일을 다운로드한다.

## 5. 기술 선택 메모

- 사용 언어:
  미정. 웹 구현과 OCR 연동이 쉬운 언어를 우선 고려
- 프레임워크:
  프론트엔드와 백엔드가 분리되거나 통합된 웹 프레임워크를 검토
- 라이브러리 후보:
  OCR 라이브러리, 이미지 전처리 도구, `PDF` 생성 라이브러리, 스프레드시트 출력 로직
- 테스트 도구:
  샘플 이미지 기반 추출 테스트, 업로드 플로우 테스트, `PDF` / `XLS` 레이아웃 검증 테스트

## 7. 검토가 필요한 설계 이슈

- [ ] 이미지마다 단어와 뜻이 어떤 형식으로 배치되는지 확인 필요
- [ ] OCR 결과에서 줄 단위 매칭으로 충분한지, 좌표 기반 분석이 필요한지 검토 필요
- [ ] 업로드 파일 크기 제한과 처리 시간 기준을 어떻게 정할지 검토 필요
- [ ] 회차별 셔플과 번호 유지 방식을 추가 옵션으로 노출할지 검토 필요
- [ ] `PDF`와 `XLS`의 정답지 포함 여부를 선택 가능하게 할지 검토 필요
- [ ] 뜻이 여러 개인 단어 또는 한 줄에 여러 항목이 있는 경우 처리 규칙 필요
