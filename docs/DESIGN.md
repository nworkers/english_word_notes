# Design Notes

이 문서는 이미지 기반 영어 단어 자료를 웹에서 업로드받아 암기 노트로 변환하기 위한 설계 초안입니다.

## 1. 목표 구조

- 애플리케이션 형태:
  웹페이지에서 여러 이미지 파일을 업로드받고, 서버 또는 백엔드 처리 후 결과를 화면에 보여주는 웹 서비스
- 주요 실행 흐름:
  이미지 업로드 -> OCR 추출 -> 단어/뜻 구조화 -> 암기 노트 생성 -> 웹 화면 출력 -> 브라우저 인쇄
- 결과물 생성 방식:
  단어 쓰기용 섹션과 뜻 쓰기용 섹션을 포함한 학습용 결과를 웹페이지에서 보여주고, 프린터 출력에 적합한 인쇄 레이아웃으로 제공

## 2. 구성 요소

- UI 또는 입력 인터페이스:
  여러 이미지 파일을 업로드할 수 있는 웹 업로드 화면
- 핵심 로직:
  OCR 결과에서 영어 단어와 뜻을 구분하고 쌍으로 매핑하는 로직
- 데이터 저장소:
  업로드된 파일 정보와 추출된 단어 목록을 메모리 또는 중간 구조 데이터로 관리
- 출력 생성기:
  학습용 노트 형식으로 재구성해 웹 화면 및 프린터 출력용 레이아웃을 만드는 렌더링 모듈

## 3. 주요 화면

- 업로드 화면:
  사용자가 여러 이미지 파일을 선택하고 업로드를 시작하는 화면
- 결과 화면:
  추출된 단어장과 암기 노트를 확인하는 화면
- 출력 화면:
  프린터로 출력하기 쉬운 형태로 단어장 내용을 정리하는 화면 또는 인쇄 전용 스타일

## 4. 데이터 흐름

1. 사용자가 웹페이지에서 `jpg` 또는 `png` 이미지 파일 여러 개를 업로드한다.
2. 업로드 처리 모듈이 파일을 수신하고 OCR 처리 대상으로 전달한다.
3. OCR 모듈이 각 이미지에서 텍스트를 읽어낸다.
4. 파서가 텍스트에서 영어 단어와 뜻의 관계를 분석한다.
5. 정규화 단계에서 공백, 중복, OCR 오인식 가능 항목을 정리한다.
6. 출력 모듈이 다음 두 섹션을 포함한 암기 노트를 생성한다.
   - 영어 단어 제시 후 뜻을 쓰는 섹션
   - 뜻 제시 후 영어 단어를 쓰는 섹션
7. 결과 화면이 생성된 단어장을 사용자에게 보여준다.
8. 인쇄 레이아웃이 프린터 출력에 맞게 적용된다.
9. 사용자는 웹에서 결과를 확인하고 프린터로 출력한다.

## 5. 기술 선택 메모

- 사용 언어:
  미정. 웹 구현과 OCR 연동이 쉬운 언어를 우선 고려
- 프레임워크:
  프론트엔드와 백엔드가 분리되거나 통합된 웹 프레임워크를 검토
- 라이브러리 후보:
  OCR 라이브러리, 이미지 전처리 도구, 문서 출력 라이브러리
- 테스트 도구:
  샘플 이미지 기반 추출 테스트, 업로드 플로우 테스트, 인쇄 레이아웃 검증 테스트

## 7. 검토가 필요한 설계 이슈

- [ ] 이미지마다 단어와 뜻이 어떤 형식으로 배치되는지 확인 필요
- [ ] OCR 결과에서 줄 단위 매칭으로 충분한지, 좌표 기반 분석이 필요한지 검토 필요
- [ ] 업로드 파일 크기 제한과 처리 시간 기준을 어떻게 정할지 검토 필요
- [ ] 출력 결과를 브라우저 인쇄 중심으로 할지 별도 파일 생성까지 할지 결정 필요
- [ ] 인쇄 시 용지 크기, 여백, 줄 간격, 페이지 나눔 기준을 어떻게 정할지 필요
- [ ] 뜻이 여러 개인 단어 또는 한 줄에 여러 항목이 있는 경우 처리 규칙 필요
