# 영단어 연습노트

영어 단어가 포함된 이미지 파일을 웹페이지에서 업로드하고, 이를 바탕으로 학습용 암기 노트를 생성한 뒤 `PDF` 또는 `XLS`로 내려받을 수 있게 하는 웹 서비스 프로젝트입니다.

## 프로젝트 문서

- [사양 명세](docs/PROJECT_SPEC.md)
- [설계 문서](docs/DESIGN.md)
- [작업 기록](docs/work-logs/README.md)

## 프로젝트 개요

- 웹페이지에서 여러 개의 `jpg` 또는 `png` 이미지 파일을 업로드할 수 있습니다.
- 업로드된 이미지 안의 영어 단어와 뜻을 읽어 단어-뜻 쌍으로 정리합니다.
- 정리된 내용을 바탕으로 웹에서 단어장을 확인하고 `PDF` 또는 `XLS`로 내려받을 수 있습니다.
- 암기 노트는 다음 두 가지 형식을 모두 포함합니다.
  - 단어만 보고 뜻을 쓰는 형식
  - 뜻만 보고 단어를 쓰는 형식
- 출력 결과는 학습자가 직접 손으로 적을 수 있도록 `A4` 인쇄를 고려한 표 형태로 구성됩니다.
- 각 학습 유형은 반복 회차를 따로 지정할 수 있으며, `0`회로 두면 해당 유형은 결과에서 제외됩니다.
- 마지막에는 원본 추출 순서 기준의 정답지가 함께 포함됩니다.

## 추출 메모

- 현재 추출은 Vision 모델이 이미지에서 단어와 뜻을 직접 읽는 방식으로 동작합니다.
- `Ollama`, `Gemini`, `OpenAI` 중 원하는 공급자를 선택할 수 있습니다.
- 번호가 보이는 원본은 번호를 우선 보존하고, 챕터 재시작이 있을 때만 이어지는 번호로 보정합니다.

## 내보내기 메모

- `PDF`는 흑백 `A4` 표 형식으로 생성됩니다.
- `PDF` 상단 제목은 추출 결과를 바탕으로 자동 생성되며, 결과 화면에서 직접 수정할 수 있습니다.
- 문제지는 `번호 | 문항 | 답안` 구조를 사용하며 번호는 원본 추출 순서를 따릅니다.
- 원본 이미지에 인쇄된 번호가 있으면 그 번호를 우선 사용하고, 챕터별로 번호가 다시 시작되면 뒤 차시는 이어지는 번호로 자동 보정합니다.
- 화면의 `구조화된 추출 결과`는 번호순 JSON으로 표시되고, `정답지`는 별도 번호표 섹션으로 확인할 수 있습니다.
- 현재 결과를 JSON 파일로 저장해 두었다가, 나중에 다시 업로드해 Vision 추출을 재실행하지 않고 결과를 복원할 수 있습니다.
- `XLS`도 동일한 구조를 따르며 인쇄 시 첫 행 반복과 `A4` 맞춤 설정을 포함합니다.
- 정답지는 문서 마지막 또는 마지막 시트에 `번호 | 단어 | 뜻` 형태로 포함됩니다.

## Ollama 연동

- 설정 팝업에서 `Ollama`, `Gemini`, `OpenAI` 설정을 브라우저 로컬 저장소에 저장할 수 있습니다.
- `Ollama`는 로컬 서버뿐 아니라 `https://ollama.com/api`와 API Key를 사용한 `Ollama Cloud`도 지원합니다.
- `Gemini`와 `OpenAI`는 브라우저에 저장된 API Key와 모델 설정을 사용해 Vision 추출을 수행할 수 있습니다.
- 모델 프롬프트의 예외 규칙은 뜻 영역에만 적용되며, 단어 영역의 표제어는 제거 대상에서 제외합니다.
- 기본값은 `Ollama Vision only`이며, 아래 환경 변수를 설정하면 로컬 `Ollama` 기본값도 조정할 수 있습니다.

```bash
OLLAMA_ENABLED=true
OLLAMA_BASE_URL=http://127.0.0.1:11434
OLLAMA_MODEL=gemma4:e2b
OLLAMA_VISION_MODEL=gemma4:e4b
```

- Vision 모드는 각 공급자의 이미지 입력 지원 모델이 설정되어 있어야 동작합니다.
